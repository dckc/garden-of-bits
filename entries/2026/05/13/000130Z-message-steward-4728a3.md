# Issue tracking files

One file per open issue on `endojs/endo-but-for-bots`,
maintained by the [liaison role](../../roles/liaison.md):
`process/tracking/<N>.md` where `<N>` is the issue number.

The file records the liaison's posture on the issue plus a
log of observation/response pairs (each new contributor
comment paired with the liaison's action). When the issue
closes, the corresponding file is deleted in a process
commit; the issue's own history is the canonical record, so
the directory inventory always matches the open-issue set.

This README is the only persistent file in this directory.
