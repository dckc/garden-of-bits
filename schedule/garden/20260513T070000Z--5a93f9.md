# Daily midnight Pacific progress summary

A periodical that fires every day at 00:00 America/Los_Angeles (DST-aware), dispatched by the [timekeeper](../../../../<garden-root>/roles/timekeeper/AGENT.md) into a [journalist](../../../../<garden-root>/roles/journalist/AGENT.md) subagent. The journalist reads journal entries from the prior 24 hours, summarizes progress across every project (not just one), and writes the summary as a periodical at `journal/periodicals/<YYYY>/<MM>/<DD>.md` (per the periodicals tree's layout in `journal/periodicals/README.md`).

Scope is intentionally everything: dispatches, results, ticks, messages, worktree-lifecycle entries. The journalist's `daily-progress-summary` purpose handles partitioning the input by project and by activity kind. See the journalist's role file for the periodical's shape.

Recurrence is `daily-at-00:00-America/Los_Angeles`. The next trigger is always computed forward from the *intended* schedule (the prior trigger plus one local day), via `<garden-root>/skills/scheduling/next-trigger.sh`. This means a late firing does not shift the daily anchor; the trigger remains pinned to local-midnight Pacific even across DST transitions and across cycle latency.

This file is the seed scheduled event. The timekeeper's first cycle (gated on the bulletin's *Awaits maintainer decision* row) will pick it up; on firing, the timekeeper writes the next day's event file (e.g. `20260514T070000Z--<new-short-id>.md`) before archiving this one under `_fired/garden/`.
